# Living Blueprint

*The single-document, 9-layer architecture of this commons. This is the most important document in the repository — it describes what this system IS and what it should BECOME.*

A Blueprint is a **temporal specification**: a structured description of the living system at multiple time horizons simultaneously. It holds a baseline (the system as it exists today), a near-term timeslice (12-24 months), and a long-horizon timeslice (3-5+ years). None of these are predictions. They are design targets that the community commits to engineering toward.

The work of moving between timeslices is **transition engineering**: sensing current state, identifying the gap against the next timeslice, designing the transition path using proven patterns, and building the next state deliberately.

**How to use this document:**
- Start with L1. Do not proceed until L1 is articulated clearly.
- Each layer builds on the ones above it. L1 constrains everything below.
- The agent helps you fill this in during boot. Challenge it. Argue. Iterate.
- This is a living document — it will be rewritten as the commons grows.
- Target: 20-120+ pages when fully populated. Thin at boot, rich at maturity.
