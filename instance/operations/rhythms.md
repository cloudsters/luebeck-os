# Operating Rhythms

*The recurring rhythms that keep your commons alive and aligned. Define your rhythms here — start with the defaults below and adapt as your commons matures.*

---

## Default Rhythms

| Rhythm | Frequency | Who | What happens |
|---|---|---|---|
| **Alignment check** | Weekly | Purpose Agent | Run `ALIGN.md` checks, post results as GitHub Issue, flag critical items |
| **Board review** | Monthly | All agents + founder | Review alignment reports, set priorities, course-correct, update blueprint |
| **Blueprint review** | Quarterly | Purpose Agent + founder | Review and update the Living Blueprint (L1-L9), assess timeslice progress |
| **Upstream sync review** | When PR arrives | Purpose Agent | Review automated upstream sync PR, merge or discuss changes |

## Stage-Specific Rhythms

Add rhythms as your commons grows:

| Stage | Additional rhythms to consider |
|---|---|
| **Seed** | Weekly founder check-in, pattern application reviews |
| **Active** | Touchpoint health checks, value stream retrospectives |
| **Gathering** | Community event planning, stakeholder feedback collection |
| **Established** | Governance reviews, resilience assessments, contribution planning |
| **Mature** | Upstream contribution cycles, mentoring new commons |
